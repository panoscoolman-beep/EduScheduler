# Ιστορικό Αλλαγών & Αναβαθμίσεων EduScheduler

Το παρόν έγγραφο συνοψίζει όλες τις παρεμβάσεις, διορθώσεις και νέες λειτουργίες που αναπτύχθηκαν στο σύστημα **EduScheduler**, στοχεύοντας στην πλήρη προσαρμογή του για χρήση από φροντιστήρια, καθώς και στην αυτοματοποίηση της φιλοξενίας και της συντήρησής του.

## 1. Υποδομή, Deployment & Ασφάλεια Δεδομένων (Production / Server)
- **Μετεγκατάσταση στον Debian Server:** Η εφαρμογή μεταφέρθηκε από το τοπικό περιβάλλον στον δικό σας Debian-based mini-PC (port `8082`), σε ξεχωριστό περιβάλλον (Docker Network) ώστε να μην εμπλέκεται με άλλες εφαρμογές (π.χ. korifi-crm).
- **Αυτοματοποίηση CI/CD (GitHub Actions):** Στήθηκε ένας self-hosted runner. Σε κάθε `git push` στο Github, ο server λαμβάνει αυτόματα τον νέο κώδικα, κάνει rebuild το Docker Container και επανεκκινεί την εφαρμογή χωρίς να απαιτείται καμία χειροκίνητη ενέργεια.
- **Αυτοματοποιημένα Backups (Rclone):** Ενημερώθηκε το κεντρικό script δημιουργίας αντιγράφων ασφαλείας του server (`/home/coolman/scripts/backup_all.sh`). 
  - Πλέον κάνει **Live Dump** της βάσης PostgreSQL του EduScheduler μέσα από το Container.
  - Συμπιέζει ολόκληρο τον φάκελο εφαρμογής & τα δεδομένα.
  - Τα συγχρονίζει ασύγχρονα με το Google Drive για τη διασφάλιση των δεδομένων.

## 2. Αρχιτεκτονική Πολλαπλών Εγγραφών (Many-to-Many) για Φροντιστήρια
Στα ιδιωτικά φροντιστήρια, σε αντίθεση με τα δημόσια σχολεία, ένας μαθητής δεν ανήκει σε ένα μόνο κλειστό τμήμα, αλλά μπορεί να παρακολουθεί πολλαπλά τμήματα (πχ. "Αλγεβρα Β Λυκείου" και "Έκθεση Β Λυκείου" ή τμήματα επιλογής).
- **Βάση Δεδομένων (Models):** Προστέθηκαν τα νέα μοντέλα `Student` (Μαθητές) και `StudentClassEnrollment` (Εγγραφές). Πλέον δημιουργείται μια δομή γέφυρας όπου **Ν Μαθητές** μπορούν να ανήκουν σε **Μ Τμήματα**.
- **Νέο API & Backend Logic:** Κατασκευάστηκε ολόκληρο το σύστημα υποδοχής, αποθήκευσης και επιστροφής αυτών των δεδομένων (`backend/routers/students.py` & επεμβάσεις στο `classes.py`).

## 3. Έξυπνος Αλγόριθμος (Solver Engine) 
- **Hard Constraint H7 (Αποφυγή Επικάλυψης Μαθητή):** Ο αλγόριθμος τεχνητής νοημοσύνης του EduScheduler (Google OR-Tools / CP-SAT) αναβαθμίστηκε με έναν πολύπλοκο κανόνα. 
- Όταν ο αλγόριθμος αποφασίζει τη θέση δύο διαφορετικών υποχρεώσεων/τμημάτων, ελέγχει αν αυτά τα δύο τμήματα **μοιράζονται έστω και έναν (1) κοινό εγγεγραμμένο μαθητή**. 
- Εάν ισχύει κάτι τέτοιο, **απαγορεύει αυστηρά** στην Τεχνητή Νοημοσύνη να προγραμματίσει αυτά τα μαθήματα την ίδια ώρα και μέρα, δημιουργώντας έτσι 100% εφαρμόσιμα προγράμματα που σέβονται τις ώρες όλων των μαθητών.

## 4. Περιβάλλον Διεπαφής χρήστη (Frontend UI)
- **Καρτέλα "Μαθητές" (🎓):** Προστέθηκε νέα αυτόνομη οθόνη διαχείρισης μαθητών, όπου μπορείτε να προσθέσετε, επεξεργαστείτε και να αφαιρέσετε το πελατολόγιό σας (Ονοματεπώνυμα, Email, Τηλέφωνα).
- **Καρτέλα "Τάξεις":** Η φόρμα δημιουργίας και επεξεργασίας κάθε τμήματος αναβαθμίστηκε ριζικά. Αντί για μία απλή επιλογή "Αριθμού Μαθητών", υπάρχει πλέον ένα πεδίο **πολλαπλής επιλογής (Multi-Select)**, μέσα από το οποίο βλέπετε την πλήρη λίστα των μαθητών σας και επιλέγετε (με Ctrl/Cmd + Κλικ) ποιοι συγκεκριμένα μαθητές συμμετέχουν στο τμήμα.
- **Επανασχεδιασμός κεντρικού Dashboard:** Η οθόνη εκκίνησης ("Γρήγορη Εκκίνηση") ενημερώθηκε και επανασχεδιάστηκε για να αντανακλά τη νέα λογική σε 7 βασικά βήματα υποχρεώσεων, συμπεριλαμβάνοντας πλέον και τη διεκπεραίωση προσθήκης μαθητών. Επιπλέον, προστέθηκε ένα "Stat Card" για να βλέπετε άμεσα το συνολικό πλήθος των εγγεγραμμένων μαθητών.

## 5. Επιδιορθώσεις Σφαλμάτων & Υποστήριξη (Hotfixes)
- Αντιμετωπίστηκε πρόβλημα `404 Not Found` στο API, προσαρμόζοντας σωστά τα prefixes των HTTP Routers για την οθόνη Μαθητών και Τάξεων.
- Διορθώθηκε σφάλμα κωδικοποίησης χαρακτήρων (Null Bytes / UTF-16) που προκάλεσε το Windows PowerShell κατά τη συγγραφή ενός αρχείου. Αναταύτηκε σωστά το Docker Container, επιβεβαιώνοντας παράλληλα και την ανθεκτικότητα / ανοχή της εφαρμογής σε fail-states.
